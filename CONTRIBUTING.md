# Contributing

Thanks for your interest in improving this example. It is a reference app for
the [Privasys](https://privasys.org) confidential-computing platform, kept
deliberately small so developers can read it end to end and copy it as a
starting point for their own confidential app.

## Ground rules

- Keep it minimal. This repo exists to teach one thing well; please avoid
  changes that obscure the core example.
- Match the existing style and keep pull requests focused.
- By contributing, you agree that your contributions are licensed under the
  MIT License (see [LICENSE](LICENSE)).

## Getting started

1. Fork and clone the repo.
2. Follow the [README](README.md) to build and run the app locally.
3. Make your change and confirm the app still builds and runs.

## Submitting a change

- Open a pull request with a clear description of what changed and why.
- Keep CI green: pushes and pull requests build/test the app. On the container
  examples, pushing a `v*` tag also publishes the image and cuts a release.

## Security

Please do not report security issues in public pull requests or issues. See
[SECURITY.md](SECURITY.md) for how to disclose vulnerabilities privately.
